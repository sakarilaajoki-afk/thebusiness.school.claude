A one-page reference for your school's IT or network team to whitelist The Business School Simulation. All connections are over standard HTTPS — no special protocols, no plugins, no installs. Forward this to IT and they should be able to whitelist in under 5 minutes.

Protocols
HTTPS · port 443   WSS · port 443
Standard HTTPS (TLS 1.2+) plus secure WebSocket for real-time classroom session state. No custom ports.
Browser
Modern browsers: Chrome, Edge, Firefox, Safari (last 3 versions)
No installs, no plugins, no extensions, no admin rights required.
Devices
School-managed Windows laptops · Chromebooks · iPads · iMacs · BYOD
Browser-based — runs anywhere with a modern browser. No app to manage.
Data & Privacy
No student personal data collected. Students join via teacher PIN only — no email, no account, no name. Sessions ephemeral by default; teacher dashboard data hosted in EU (Netlify Frankfurt). UK GDPR registered.
Safeguarding (KCSIE 2024/25)
No chat. No messaging. No user-generated content. Students interact only with the simulation's market mechanics; no peer-to-peer communication. Teacher controls all session triggers and end conditions.

What gets sent over the network

  • Outbound from school: teacher login + PIN generation (small JSON payloads); student decisions during rounds (a few KB per student per round)
  • Inbound to school: session state updates over WebSocket (text only, no media); end-of-session AI-generated feedback PDF (~50–200 KB per student)
  • No video, no audio, no large media transfers — total bandwidth per 30-student session is typically under 5 MB across the full session
  • No third-party trackers or ads — the simulation itself does not load external advertising or social-media scripts
